# Reusable Workflow

The reusable workflow is located at
`.github/workflows/protected-file-review.yml` and is called by a workflow in a
consuming repository.

Private repository access must be enabled in the `abuzucom` organization for
the consuming repository to use this workflow. Prefer a reviewed release tag
such as `v1`; pinning the `uses` reference to a commit SHA provides stronger
provenance.

The caller owns the event trigger and permissions. Keep the caller workflow on
the default branch and grant only:

```yaml
permissions:
  contents: read
  pull-requests: read
```

The called workflow checks out the default branch of the caller repository and
runs its trusted checker. It does not use `github.head_ref`, a PR ref, or an
artifact generated by the PR.
